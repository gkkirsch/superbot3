How It Works

Built on Claude Code's Internals

Every architectural decision maps to how Claude Code actually works under the hood.

✉

Inbox Messaging

Agents communicate via JSON inbox files with lockfile-protected writes. The same protocol Claude Code uses for teammate coordination, extended across spaces.

⚙

CLAUDE_CONFIG_DIR

One env var isolates everything: plugins, skills, agents, teams, settings, inboxes. Each space gets its own config directory. Auto-propagates to workers.

★

Compiled Knowledge

Karpathy-style wiki compiled from raw sources. Drop docs in raw/, the consolidator builds summaries, concepts, and cross-references. Query it like a database.

☀

Persistent Memory

MEMORY.md always in the system prompt. Topic files for decisions, preferences, patterns. Nightly consolidation reads transcripts and extracts what matters.

⬥

Autonomous Spaces

Each space runs as a Claude Code swarm leader. Spawns workers, manages schedules, writes its own skills. The master orchestrator just keeps them alive.

↻

Self-Improving

Agents edit their own CLAUDE.md, create skills for repeated tasks, log learnings. The 3-occurrence rule: patterns that repeat get promoted to permanent behavior.

⏳

Broker Scheduler

Node.js broker reads scheduled_tasks.json every 30 seconds and fires prompts to space inboxes. Nightly memory + knowledge consolidation runs automatically.

⚖

Plugin Ecosystem

Install plugins per space from the marketplace. Skills, hooks, agents, MCP servers. Credentials isolated via keychain with per-space hashing.

Getting Started

Three commands to a running AI system.

1

Install

One command installs everything — broker, dashboard, CLI, skills, and plugins system.

2

Create a Space

Each project gets its own space with isolated Claude Code instance, plugins, knowledge, and memory.

3

Start Building

Chat with spaces from the dashboard or CLI. Workers execute tasks, escalate what they can't decide.
